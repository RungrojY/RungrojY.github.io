1 Setup Hadoop Cluster

1.1 Create virtual machine

Create Ubuntu Server image with yaml file.

hostname role
node0 namenode
node1 datanode
node2 datanode 
setup-ubuntu.yaml
package_update: true
package_upgrade: true
packages:
  - openjdk-8-jdk
  - ssh
Window Terminal
# the default resources are 1 CPU cores, ~1 GB memory and ~5 GB disk space.
multipass launch --name node0 --network Wi-Fi --cloud-init C:\Users\Jorgnur\Desktop\Clound\multipass\setup-ubuntu.yaml focal
# or add more resources with the command
multipass launch -n node0 --network Wi-Fi --cloud-init C:\Users\Jorgnur\Desktop\Clound\multipass\setup-ubuntu.yaml -c 1 -d 10G -m 4G focal
multipass launch -n node1 --network Wi-Fi --cloud-init C:\Users\Jorgnur\Desktop\Clound\multipass\setup-ubuntu.yaml -c 1 -d 10G -m 2G focal
multipass launch -n node2 --network Wi-Fi --cloud-init C:\Users\Jorgnur\Desktop\Clound\multipass\setup-ubuntu.yaml -c 1 -d 10G -m 2G focal

The word focal refers to Ubuntu 20.04 LTS. The IP addresses are obtain as

hostname role IP address
node0 namenode 192.168.1.127
node1 datanode 192.168.1.128
node2 datanode 192.168.1.129

1.4 Install pdsh (Parallel Distributed Shell) in node0

For executing shell commands in parallel across multiple remote machines, let’s install pdsh and it is required for Hadoop setup

sudo apt-get install pdsh

# Display output version information with
pdsh -V

Please change the type of shell protocol to be ssh, otherwise it’s cause an error for Hadoop.

vi ~/.bashrc
.bashrc
export PDSH_RCMD_TYPE=ssh

By testing with the following command, the hostname must be show up.

pdsh -w node[0-2] hostname

or testing with the text file.

vi hosts.txt

Set all IP address of machine nodes

hosts.txt
192.168.1.127
192.168.1.128
192.168.1.129
pdsh -w ^hosts.txt hostname

1.5 Prepare the file

Let’s visit released page and download hadoop-3.3.5.tar.gz in Window

Window Terminal
multipass transfer C:\Users\Jorgnur\Downloads\hadoop-3.3.5.tar.gz node0:/home/ubuntu/

In Ubuntu, unpacking the software on all the machines in the cluster

tar -zxvf ~/Downloads/hadoop-3.3.5.tar.gz

1.6 Setup .bashrc

Edit .bashrc with

vi ~/.bashrc
export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64
export PATH=$PATH:$JAVA_HOME/bin
export HADOOP_HOME=~/hadoop-3.3.5
export PATH=$PATH:$HADOOP_HOME/bin
export PATH=$PATH:$HADOOP_HOME/sbin

reload bash file

source ~/.bashrc
# or
. ~/.bashrc

# check the result with
echo $PATH

1.7 Hadoop Configuration

Let’s add JAVA_HOME to flies located in ~/hadoop-3.3.5/etc/hadoop which are

filename
mapred-env.sh
hadoop-env.sh
yarn-env.sh 
cd ~/hadoop-3.3.5/etc/hadoop

# see file list in details
ls -lstr
ls -lstr *env*.sh

Open three files with the following commands

vi ~/hadoop-3.3.5/etc/hadoop/mapred-env.sh
vi ~/hadoop-3.3.5/etc/hadoop/hadoop-env.sh
vi ~/hadoop-3.3.5/etc/hadoop/yarn-env.sh

and add JAVA_HOME below to all files.

export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64

1.8 Configure core-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/core-site.xml
<!-- Set master node -->
<property>
  <name>fs.default.name</name>
  <value>hdfs://192.168.1.127:50000</value>
</property>

1.9 Configure yarn-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/yarn-site.xml
<property>
  <name>yarn.nodemanager.aux-services</name> <value>mapreduce_shuffle</value>
</property>

<property>
  <name>yarn.nodemanager.aux-services.mapreduce.shuffle.class</name> 
  <value>org.apache.hadoop.mapred.ShuffleHandler</value>
</property>

<property>
  <description>The hostname of the RM.</description>
  <name>yarn.resourcemanager.hostname</name>
  <value>192.168.1.127</value>
</property>

<property>
  <description>The address of the applications manager interface in the RM.</description>
  <name>yarn.resourcemanager.address</name>
  <value>192.168.1.127:8032</value>
</property>

1.10 Configure hdfs-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/hdfs-site.xml
<property>
  <name>dfs.namenode.name.dir</name>
  <value>/home/ubuntu/hadoop-data/namenode-dir</value>
</property>

<property>
  <name>dfs.datanode.data.dir</name>
  <value>/home/ubuntu/hadoop-data/datanode-dir</value>
</property>

1.11 Configure mapred-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/mapred-site.xml
<property>
  <name>mapreduce.framework.name</name>
  <value>yarn</value>
</property>

2.2 NameNode Web Interface

The web interface for the NameNode is available at http://localhost:9870/ by default. If the ufw is enable, you should allow add ufw rule on port 9870

sudo ufw allow 9870
sudo ufw reload

2.4 Obtain public key of each

Using pdsh and regular expression to retreive the rsa public key of each machine

pdsh -w node[1-2] cat ~/.ssh/id_rsa.pub | sed 's/[^:]*: //' >> text.txt

2.5 Configuration files

Hadoop’s Java configuration is driven by two types of important configuration files

Read-only default configuration

hadoop-3.3.5\share\doc\hadoop\hadoop-project-dist\hadoop-common\core-default.xml
hadoop-3.3.5\share\doc\hadoop\hadoop-project-dist\hadoop-hdfs\hdfs-default.xml
hadoop-3.3.5\share\doc\hadoop\hadoop-yarn\hadoop-yarn-common\yarn-default.xml
hadoop-3.3.5\share\doc\hadoop\hadoop-mapreduce-client\hadoop-mapreduce-client-core\mapred-default.xml

Site-specific configuration

hadoop-3.3.5/etc/hadoop/core-site.xml
hadoop-3.3.5/etc/hadoop/hdfs-site.xml
hadoop-3.3.5/etc/hadoop/yarn-site.xml
hadoop-3.3.5/etc/hadoop/mapred-site.xml

HDFS daemons are

NameNode
SecondaryNameNode
DataNode

YARN daemons are

ResourceManager
NodeManager
WebAppProxy
