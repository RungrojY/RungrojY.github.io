1 Prepare VM for duplication

1.1 Install Virtual Machine

For example, I will use Ubuntu Server 20.04.6 in this setup. Install it in Virtual Box and don’t forget to install ssh I have set CPU 2 cors, Memory 4 GB and Space 25 GB. By default, the network is set as NAT, so that the VM can use internet.

1.2 Install Java

Install java version 8 for Hadoop. This is the required java version.

sudo apt update
sudo apt install openjdk-8-jdk

The next is to set the variables to ~/.bashrc. To add a literal string to the ~/.bashrc file in Ubuntu, you can use the echo command with single quotes to prevent variable expansion.

echo 'export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
# reload
source ~/.bashrc
# check
echo $JAVA_HOME

Using single quotes (') ensures that the string is treated as a literal and preserves the $PATH variable for future evaluation when the .bashrc file is sourced.

1.3 Download Hadoop

Let’s visit released page and download hadoop-3.3.5.tar.gz or using the following command.

wget https://dlcdn.apache.org/hadoop/common/hadoop-3.3.5/hadoop-3.3.5.tar.gz

Extract the file with the command:

tar -zxvf hadoop-3.3.5.tar.gz
# remove a tar file
rm hadoop-3.3.5.tar.gz

In order to run the Hadoop command without specifying the directory, let add the variables. Assume that the directory of hadoop is /home/ubuntu/hadoop-3.3.5, you can use:

echo 'export HADOOP_HOME=/home/ubuntu/hadoop-3.3.5' >> ~/.bashrc
echo 'export PATH=$PATH:$HADOOP_HOME/bin' >> ~/.bashrc
echo 'export PATH=$PATH:$HADOOP_HOME/sbin' >> ~/.bashrc
# reload
source ~/.bashrc
# check
echo $HADOOP_HOME

1.4 Hadoop Configuration

Let’s add JAVA_HOME to flies located in ~/hadoop-3.3.5/etc/hadoop which are

filename
mapred-env.sh
hadoop-env.sh
yarn-env.sh 
cd ~/hadoop-3.3.5/etc/hadoop

# see file list in details
ls -lstr
ls -lstr *env*.sh

Open three files with the following commands

vi ~/hadoop-3.3.5/etc/hadoop/mapred-env.sh
vi ~/hadoop-3.3.5/etc/hadoop/hadoop-env.sh
vi ~/hadoop-3.3.5/etc/hadoop/yarn-env.sh

and add JAVA_HOME below to all files.

export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64

1.5 Configure core-site.xml

In the this configuration, you need to have the IP address of a VM. You can add the second network as host-only adapter in Virtual Box of the machine to obtain the IP address.

vi ~/hadoop-3.3.5/etc/hadoop/core-site.xml

Let the machine with a IP address 192.168.1.127 be the namenode. Paste the following code in <configuration>...</configuration>

<property>
  <name>fs.defaultFS</name>
  <value>hdfs://192.168.1.127:9000</value>
</property>

1.6 Configure hdfs-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/hdfs-site.xml

By default, data replication is 3, thus add the property dfs.replication to define more or less. Then, define the location to store hdfs files of namenode and datanode

<property>
  <name>dfs.replication</name>
  <value>2</value>
</property>

<property>
  <name>dfs.namenode.name.dir</name>
  <value>/home/ubuntu/hadoop-data/namenode</value>
</property>

<property>
  <name>dfs.datanode.data.dir</name>
  <value>/home/ubuntu/hadoop-data/datanode</value>
</property>

1.7 Configure mapred-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/mapred-site.xml

Tell Hadoop to use yarn to manage mapreduce.

<property>
  <name>mapreduce.framework.name</name>
  <value>yarn</value>
</property>

1.8 Configure yarn-site.xml

vi ~/hadoop-3.3.5/etc/hadoop/yarn-site.xml
<property>
  <name>yarn.nodemanager.aux-services</name> 
  <value>mapreduce_shuffle</value>
</property>

<property>
  <name>yarn.resourcemanager.hostname</name>
  <value>192.168.1.127</value>
</property>

3.3 NameNode Web Interface

The web interface for the NameNode is available at http://192.168.1.127:9870/ by default. Normally, the ufw is inactive after install Ubuntu image. you do not have to do here. If the ufw is enable for some cases, you should allow add ufw rule on port 9870

sudo ufw allow 9870
sudo ufw reload

3.5 How to get different IP on cloned machines ?

Firstly, check what the ip address on a VM.

ip a

I figure out how to use NetworkManager to handle IP address in User-host Adapter Network Settings. You can check the status of the NetworkManager service by running the following command:

systemctl status NetworkManager

If it is not available, install with commands

sudo apt update
sudo apt install network-manager
# check it again
systemctl status NetworkManager
# or start it manually
sudo systemctl start NetworkManager

Then, change yaml in /etc/netplan

cd /etc/netplan
ls

Open the yaml file, e.g., cat 00-installer-config.yaml

# This is the network config written by 'subiquity'
network:
  ethernets:
    enp0s3:
      dhcp4: true
  version: 2

and chang it into

# Let NetworkManager manage all devices on this system
network:
  version: 2
  renderer: NetworkManager
sudo netplan try
sudo netplan apply
# check the IP
ip a
